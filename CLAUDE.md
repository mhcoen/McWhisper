# Visual verification

Use `appshot` to capture screenshots of the running application.
`appshot` captures a macOS app window by process name. It works
with any app that puts a window on screen.

```bash
appshot "AppName" screenshots/current/main.png
appshot "AppName" screenshots/current/main.png --launch ./run.sh
appshot "AppName" screenshots/current/main.png --wait 2
```

Always use appshot for visual verification. Do not reinvent
screenshot capture with other approaches.

After capturing, examine the screenshot to confirm:
- The window renders correctly with expected content
- Layout and spacing are reasonable
- No blank screens, missing elements, or rendering artifacts
- Text is readable and properly styled

Do not delete `screenshots/reference/` -- those are the
target images duplo uses for visual comparison.

# Swift SPM + SwiftUI

SPM executables using SwiftUI do not automatically show windows.
Every SwiftUI app built with SPM must include this in the App
struct init:

```swift
init() {
    NSApplication.shared.setActivationPolicy(.regular)
    NSApplication.shared.activate(ignoringOtherApps: true)
}
```

Use `swift build --disable-sandbox` and `swift test --disable-sandbox`
inside Claude Code's sandbox.

# Debugging

When something crashes or behaves unexpectedly, find and read
the actual error output first. Check crash reports
(~/Library/Logs/DiagnosticReports/ on macOS), stderr, log
files, tracebacks. Do not guess from source code alone.
After fixing, reproduce the failure and verify it is gone.

# Source files

- `Package.swift` — SPM package manifest. macOS 14+, Swift 5.10. Single executable target `McWhisper` with a test target `McWhisperTests`.
- `Sources/McWhisper/McWhisperApp.swift` — App entry point (`@main`). Sets `.regular` activation policy (SPM SwiftUI workaround), then switches to `.accessory` to hide from Dock. Shows a `MenuBarExtra` with a `MenuBarLabel` view that displays a waveform icon normally and a pulsing red record icon (`record.circle`) while `AudioEngine.isRecording` is true (driven by `TimelineView` with cosine opacity pulse, period `pulsePeriod` = 1 s). Contains `StatusView` displaying coordinator state (idle/recording/transcribing/error with partial text). Uses `@NSApplicationDelegateAdaptor` with `AppDelegate`; on first launch (`!AppSettings.hasCompletedOnboarding`) shows the onboarding sheet via `OnboardingWindowController`; on subsequent launches requests microphone permission and shows an `NSAlert` if denied, checks Accessibility permission and shows an alert linking to System Settings if not granted. Owns a `@StateObject RecordingCoordinator` started via `.task` on `StatusView`.
- `Sources/McWhisper/MenuBarView.swift` — `MenuBarView` struct composing the `MenuBarExtra` menu content: `StatusView` (coordinator state), a `ModeSelectorView` (radio group of all `TranscriptionMode` modes with checkmark on selected, bound to `AppSettings.selectedMode`), "Recording History…" button (opens `HistoryWindowController`), "Settings…" button (opens `SettingsWindowController`), and "Quit McWhisper" (Cmd-Q).
- `Sources/McWhisper/HistoryView.swift` — `HistoryView` struct displaying a searchable `List` of `TranscriptionRecord` sorted by date descending in an `HSplitView` with a detail pane. Supports multi-select via `Set<UUID>` selection binding. Search filters on processed text (falls back to raw text). Each row is a `HistoryRow` showing first line of text, timestamp (abbreviated month/day/hour/minute), duration (e.g. "42s" or "2m 5s"), and a `ModeBadge` (capsule-shaped label with mode display name). Shows placeholder when store is empty. Accepts optional `onRetranscribe` callback. When records are selected, a Delete button appears below the list showing the count; tapping it presents a confirmation alert that calls `historyStore.deleteRecords(ids:)` and clears the selection. On single selection, shows `HistoryDetailView` with full processed text, `HistoryTextToggle` (Raw/Processed toggle when both differ), Copy button (writes displayed text to pasteboard), and Re-transcribe button (visible when audio file exists, invokes `onRetranscribe`). Multi-select shows a count placeholder in the detail pane. `HistoryDetailView` accepts a `TranscriptionRecord` and optional callback. `HistoryTextToggle` is a compact Processed/Raw toggle bound to `@State showRaw`.
- `Sources/McWhisper/HistoryWindowController.swift` — `HistoryWindowController` singleton (`@MainActor`) managing an `NSWindow` for recording history display. `show(historyStore:onRetranscribe:)` creates or brings forward a 600x400 window containing `HistoryView`, passing the optional re-transcribe callback. Shows placeholder text when empty.
- `Sources/McWhisper/SettingsWindowController.swift` — `SettingsWindowController` singleton (`@MainActor`) managing an `NSWindow` for settings display. `show()` creates or brings forward a 520x400 window containing `SettingsView`.
- `Sources/McWhisper/SettingsView.swift` — `SettingsView` struct with a `TabView` containing three tabs selected by `SettingsTab` enum (.general, .models, .modes). `GeneralSettingsTab`: hotkey recorder via `HotkeyRecorderView` (`NSViewRepresentable` wrapping `HotkeyRecorderNSView`), a visible rounded-rect field that becomes first responder on click, captures the next keyDown (keyCode + modifiers) or flagsChanged (modifier-only keys like Right Command), saves to `AppSettings`, and resigns focus. Escape cancels recording. Displays current hotkey via `HotkeyFormatter`. Launch-at-login toggle via `SMAppService.mainApp`. `HotkeyFormatter` enum provides `displayString(keyCode:modifiers:)` and `keyName(for:)` using `Carbon.HIToolbox` key codes, with named modifier keys (Right/Left Command/Shift/Option/Control, Caps Lock, Fn) and `UCKeyTranslate` for character lookup. `ModelsSettingsTab`: owns `@StateObject ModelDownloader`, active model `Picker` bound to `AppSettings.selectedModelID` with `onChange` guard that reverts selection if the chosen model is not downloaded (non-bundled and not `.downloaded`), list of `ModelRow` views showing display name, size label, bundled badge, checkmark for active model, download progress bar with percentage during download, and action buttons (download/cancel/delete) for non-bundled models. `ModelRow` accepts `downloadState`, `onDownload`, `onCancel`, `onDelete` parameters. `ModesSettingsTab`: lists built-in modes and custom modes with selection checkmarks bound to `AppSettings.selectedMode`, context menu for edit/delete on custom modes, "Add Custom Mode..." button opening `ModeEditSheet` (name + prompt fields with save/cancel). `ModeRow` is a plain button row with checkmark. `ModeEditSheet` supports both new and edit flows.
- `Sources/McWhisper/AppSettings.swift` — App-wide settings using `@AppStorage` / `UserDefaults`. Static properties for selected model ID, push-to-talk hotkey keycode + modifiers (default: Right Command, keyCode 54 + modifiers 0), selected mode, language selection ("auto" default), silence threshold (0.015 default, used by AudioEngine VAD and trim), panel position persistence (panelPositionX, panelPositionY, hasSavedPanelPosition), and hasCompletedOnboarding (false default, set true after onboarding sheet is dismissed). Includes `resetToDefaults()`.
- `Sources/McWhisper/TranscriptionMode.swift` — `TranscriptionMode` enum with cases `.voice`, `.message`, `.email`, `.note`, `.meeting`, and `.custom(name:prompt:)`. Custom `Codable` conformance: encodes built-in modes as `{"type":"voice"}` etc., custom modes as `{"type":"custom","name":...,"prompt":...}`; decodes both the new keyed format and legacy plain-string format (e.g. `"voice"`) for backward compatibility with old history records. Built-in modes via `builtIn`, custom modes saved/loaded from UserDefaults as JSON. Provides `allModes()`, `from(id:)` lookup, `displayName`, and string `id` for each case.
- `Sources/McWhisper/TranscriptionRecord.swift` — `TranscriptionRecord` struct (Codable, Identifiable, Equatable) with fields: id (UUID), date, duration, rawText, processedText, mode (TranscriptionMode), modelID, audioFileName (String?, optional for backward compat). Default initializer auto-generates id and date. Computed `audioFileURL` resolves to `~/Library/Application Support/McWhisper/Audio/<audioFileName>`. Computed `hasAudioFile` checks file existence on disk.
- `Sources/McWhisper/HistoryStore.swift` — `HistoryStore` class persisting an array of `TranscriptionRecord` to a JSON file in `~/Library/Application Support/McWhisper/history.json`. Supports add, updateRecord (replaces record by ID), deleteRecord(id:), deleteRecords(ids:) (batch delete by `Set<UUID>`, no-op on empty set), clearAll. Accepts optional directory for testing. Loads on init, saves on mutation.
- `Sources/McWhisper/AudioEngine.swift` — `ObservableObject` wrapping `AVAudioEngine` + `AVAudioInputNode` for microphone recording. `startRecording()` installs an audio tap, converts to 16-bit 16 kHz mono WAV via `AVAudioConverter`, and writes to a temp file. `stopRecording()` tears down the engine, trims leading/trailing silence from the WAV file via `trimSilence(url:frameDuration:threshold:)`, and returns the `.wav` URL. Exposes `@Published isRecording` state, `@Published audioLevel: Float` (RMS 0–1, updated per buffer on main thread for waveform UI), `@Published speechDetected: Bool` (energy-based VAD using 20 ms frame RMS with configurable threshold and 300 ms hangover), and a static `whisperFormat` (`AVAudioFormat`: 16 kHz, mono, Int16, interleaved). Static `trimSilence(url:frameDuration:threshold:)` reads the WAV into float, scans 20 ms frames from both ends to find first/last above-threshold frames, and rewrites the file in-place with only the speech region (all-silent files produce an empty WAV). Static VAD config: `vadFrameDuration` (0.020 s), `vadThreshold` (computed property reading from `AppSettings.silenceThreshold`), `vadHangoverDuration` (0.300 s). Throws `AudioEngineError` (alreadyRecording, notRecording, noInputAvailable, engineStartFailed, fileCreationFailed).
- `Sources/McWhisper/TranscriptionEngine.swift` — `ModelState` enum (unloaded, loading, loaded) and `TranscriptionEngine` protocol defining the common interface for local speech-to-text engines. Protocol requires `var modelState: ModelState { get }`, `var isLoaded: Bool { get }`, `var isModelCurrent: Bool { get }`, `loadModel() async throws`, `transcribe(audioURL:language:)`, `transcribe(audioSamples:language:)`, `transcribeStreaming(audioURL:language:onPartial:)`, and `transcribeStreaming(audioSamples:language:onPartial:)`. Constrained to `AnyObject` (class-only). Both `WhisperKitEngine` and `Qwen3ASREngine` conform.
- `Sources/McWhisper/TranscriptionService.swift` — `WhisperKitEngine` class (`ObservableObject`, `TranscriptionEngine`) wrapping WhisperKit for local speech-to-text. `loadModel()` async loads the Whisper model variant from `AppSettings.selectedModelID`; checks `ModelDownloader.modelsDirectoryPath` for locally downloaded models and passes `modelFolder` with `download: false` to WhisperKitConfig when found, otherwise falls back to WhisperKit's built-in download. `transcribe(audioURL:language:) async throws -> String` runs WhisperKit's pipeline on a WAV file; passes `language: nil` with `detectLanguage: true` when language is `nil` or `"auto"`. `transcribeStreaming(audioURL:language:onPartial:) async throws -> String` provides real-time partial results during decoding via WhisperKit's `TranscriptionCallback`; calls the `@Sendable onPartial` closure with each intermediate text string and returns the final result. Exposes `@Published modelState: McWhisper.ModelState`, `isModelCurrent` check, and `modelVariant(from:)` helper to extract variant name from model ID. Throws `TranscriptionError` (modelNotLoaded, transcriptionFailed, emptyResult).
- `Sources/McWhisper/HotkeyManager.swift` — `HotkeyManager` class (`ObservableObject`) providing global push-to-talk hotkey via `CGEventTap` (requires Accessibility permission). `start()` creates a session event tap listening for keyDown/keyUp/flagsChanged events matching the configured keycode + modifiers from `AppSettings`. `stop()` tears down the tap. `matches(_:)` checks a `CGEvent` against the configured hotkey. Exposes `@Published isKeyDown: Bool`, `onKeyDown`/`onKeyUp` closures called on the main thread, and computed `keyCode`/`modifiers` reading from `AppSettings`. Static `hasAccessibilityPermission` checks `AXIsProcessTrusted()`, `requestAccessibilityPermission()` prompts via system dialog. Swallows matched key events to prevent them reaching the active app. Re-enables the tap on system timeout. Throws `HotkeyManagerError` (accessibilityNotGranted, eventTapCreationFailed).
- `Sources/McWhisper/RecordingView.swift` — SwiftUI view displayed inside the floating recording panel, driven by `RecordingCoordinator.State`. Holds `@State showProcessed` toggle for switching between raw and processed text display. When `coordinator.hudMessage` is non-empty, displays `HudView` (clipboard icon + message) instead of state-based content. Idle state renders `StandbyWaveformView` (flat bars pulsing with sine wave). Recording state shows `RecordingStateView` with animated `WaveformView` (from WaveformView.swift) and live partial transcript. Transcribing state shows `TranscribingStateView` with a `ProgressView` spinner and partial text during streaming; once final result is available (`hasResult`), replaces spinner with `TextToggleView` (Raw/Processed toggle buttons) and displays the selected text via `displayText`. Error state shows `ErrorStateView` with warning icon and message. `TextToggleView` provides compact Raw/Processed toggle buttons with accent highlight.
- `Sources/McWhisper/WaveformView.swift` — `WaveformView` struct using SwiftUI `Canvas` to draw 30 vertical bars centered horizontally. Bar heights are proportional to values in a `[Float]` ring buffer; a `minimumLevel` floor (0.05) ensures bars are always visible. Static constants: `barCount` (30), `barSpacing` (2), `barCornerRadius` (1.5), `minimumLevel` (0.05), `standbyAmplitude` (0.04), `standbyPeriod` (2.0s). Static `standbyLevels(phase:count:)` generates uniform bar levels pulsing with a sine wave for standby mode. Also contains `WaveformBar` (single animated bar view with `scale` computed property using the shared minimum floor) and `StandbyWaveformView` (wraps `WaveformView` with a `TimelineView(.animation)` to drive the sine pulse continuously).
- `Sources/McWhisper/RecordingWindowController.swift` — `RecordingWindowController` class (`@MainActor`) managing a floating `NSPanel` for recording state display. Panel uses `.nonactivatingPanel` style mask so it does not steal focus. Window level is `.floating`, `hidesOnDeactivate` is false, and collection behavior includes `.canJoinAllSpaces` and `.fullScreenAuxiliary`. `show(coordinator:)` creates the panel with a fade-in animation (alphaValue 0→1) and restores last window position from `AppSettings` (or centers on screen if no saved position). `hide()` saves the current position to `AppSettings`, fades out (alphaValue 1→0), then removes the panel. Static `fadeDuration` (0.2s). `savePosition(_:)` and `restorePosition(_:)` handle position persistence. `isVisible` computed property, and exposes the `panel` for content configuration. Titlebar is transparent and hidden.
- `Sources/McWhisper/PasteManager.swift` — `PasteManager` class (`@MainActor`) managing frontmost-app capture and text pasting. `captureTarget()` saves `NSWorkspace.shared.frontmostApplication` to `targetApplication: NSRunningApplication?` (read-only). `clearTarget()` resets it to nil. `paste(_:) -> Bool` always writes text to `NSPasteboard.general`; returns `true` if the target app was available and Cmd+V was sent, `false` if no target was captured or the target app is terminated (text remains on clipboard). Called by `RecordingCoordinator`: `captureTarget()` on key-down, `paste(_:)` after transcription, `clearTarget()` after paste.
- `Sources/McWhisper/RecordingCoordinator.swift` — `RecordingCoordinator` class (`@MainActor ObservableObject`) orchestrating the push-to-talk flow. Owns `HotkeyManager`, `AudioEngine`, `WhisperKitEngine`, `HistoryStore`, `RecordingWindowController`, and `PasteManager`. `start()` subscribes to `audioEngine.$audioLevel` via Combine to fill a rolling buffer (`@Published levelSamples: [Float]`, size 30 via `levelBufferSize`) wrapped in `withAnimation(.linear(duration: 0.05))` for smooth waveform transitions, wires hotkey callbacks, and begins listening (requests Accessibility if needed), pre-loads the Whisper model. On key-down captures the frontmost app via `PasteManager.captureTarget()`, resets the level buffer and HUD state, starts recording via `AudioEngine`, and shows the floating panel with `RecordingView`; on key-up stops recording, runs `transcribeStreaming` for partial results, applies `ModeProcessor.process(_:mode:)` to the raw transcription, saves a `TranscriptionRecord` (with both raw and processed text, plus `audioFileName`) to history, pastes the processed text via `PasteManager.paste(_:)`, clears the paste target. If paste succeeds, hides the panel immediately; if paste fails (no target or terminated app), shows a "Copied to clipboard" HUD via `showHud(_:)` that auto-dismisses after `hudDisplayDuration` (2 s). Audio files are copied to `audioDirectory` (`~/Library/Application Support/McWhisper/Audio/`) via `saveAudioFile(from:)` before the temp file is cleaned up. `retranscribe(record:)` re-runs the current model and mode on a record's saved audio file, updates the record in `HistoryStore` via `updateRecord`. Exposes `@Published state: State` (idle/recording/transcribing/error), `@Published partialText: String`, `@Published rawText: String`, `@Published processedText: String`, `@Published hudMessage: String`, and `@Published levelSamples: [Float]`. Raw and processed texts are set after transcription and cleared on next key-down. `hasLevelSubscription: Bool` computed property exposes whether the Combine subscription to `audioEngine.$audioLevel` is active (for testing).
- `Sources/McWhisper/ModeProcessor.swift` — `ModeProcessor` enum with static `process(_:mode:)` applying rule-based formatting per `TranscriptionMode`. Voice and custom modes return raw trimmed text. Message mode applies sentence case and ensures trailing punctuation. Email mode wraps the body with greeting/sign-off placeholders. Note mode splits sentences into bullet-point lines. Meeting mode separates sentences with paragraph breaks. Helpers: `sentenceCase(_:)` capitalizes first letter; `splitSentences(_:)` uses Foundation's `.bySentences` enumeration.
- `Sources/McWhisper/ModelCatalog.swift` — `ModelInfo` struct (Identifiable, Equatable) with id, displayName, sizeLabel, isBundled fields. `ModelCatalog` enum listing all available Whisper models with `openai_whisper-base` as the single bundled default. Provides `bundledModelID`, `bundledModel`, `availableModels`, `downloadableModels`, and `model(for:)` lookup.
- `Sources/McWhisper/ModelDownloader.swift` — `ModelDownloader` class (`@MainActor ObservableObject`) managing WhisperKit model downloads from HuggingFace (`argmaxinc/whisperkit-coreml`) to `~/Library/Application Support/McWhisper/Models/`. Uses `URLSession` download tasks with `DownloadProgressDelegate` (`URLSessionDownloadDelegate`) for per-file progress tracking, aggregated across all files for overall progress. `ModelDownloadState` enum: `.notDownloaded`, `.downloading(progress:)`, `.downloaded`, `.failed(String)`. `downloadModel(_:) async throws` recursively lists files via HuggingFace tree API, downloads each with progress, and saves to model subdirectory. `deleteModel(_:) throws` removes the model directory. `listFiles(for:subpath:)` recursively queries HuggingFace API to discover all model files including nested `.mlmodelc` contents. `isModelDownloaded(_:)` checks for directory existence. `refreshStates()` syncs state for all catalog models. `RemoteFile` struct holds path and size. `ModelDownloadError` cases: invalidResponse, networkError, noFilesFound. Accepts optional `modelsDirectory` for testing. `nonisolated static let modelsDirectoryPath: String` provides the models directory path accessible from any isolation context (used by `WhisperKitEngine.loadModel()`). `DownloadProgressDelegate` is `@unchecked Sendable`.
